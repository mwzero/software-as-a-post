# Software as a Post

This repository collects practical examples of **Software as a Post**: applications, pipelines, and micro-tools built as a natural extension of an article, a blog post, or a shared idea.

The goal is to explore a different way of building software:

> üëâ not documentation *about* code ‚Äî but code *born from* content.

Each folder in this repo corresponds to a post (Medium, blog, or technical notes) and typically contains:

* source code of the experiment
* sample datasets (when needed)
* AI prompts or workflows used
* instructions for local execution
* minimal architectural notes

Examples are intentionally:

* **small**
* **reproducible**
* **local-first**
* optimized for fast learning
* designed as *building blocks*, not finished products

## Philosophy

*Software as a Post* means:

* turning an idea into something executable in hours
* using LLMs as accelerators, not black boxes
* favoring working prototypes over heavy frameworks
* merging narrative, code, and results into a single artifact

Each project starts from a concrete question (‚ÄúCan this run locally?‚Äù, ‚ÄúDo I really need a backend?‚Äù, ‚ÄúHow much code is actually required?‚Äù) and answers it with real software.
